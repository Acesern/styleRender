# Doré 版画风格设计说明

本文档记录 `DoreShaders.metal` 的设计思路、关键决策和技术细节，供后续迭代参考。

## 风格目标

模拟 Gustave Doré 为但丁《神曲》（1857）创作的铜版画风格：

- 密集、方向性强的交叉阴影线（cross-hatching）
- 亮部线条极细甚至近乎消失，暗部多层叠加、线条粗而密
- 硬朗的轮廓描边
- 米白/羊皮纸底色，带轻微纸张颗粒

## 渲染管线位置

与 Obra Dinn 风格共用同一套 SceneKit 两 pass 管线：

```
SceneKit DRAW_SCENE → sceneColor（rgba8）+ sceneDepth（depth32f）
                    ↓
Metal DRAW_QUAD → doreFragment → 最终画面
```

切换逻辑在 `ContentView.swift` 的 `Coordinator.update()` 中：检测到 `renderStyle` 变化时重建 `SCNTechnique`，调用 `makeDoréTechnique()` 换入新的 fragment shader。

## 核心设计原则

### 固定角度，动态线宽

**关键决策**：线条使用固定角度，不跟随曲面方向动态变化。

曾尝试用屏幕空间亮度梯度（Structure Tensor）推断每个像素的曲面走向，让线条沿等照线方向排列。但这种方案有根本性缺陷：

- 在平坦面和高光区，梯度接近零，方向信号是噪声，任何滤波都无法解决"无信号区域给出噪声方向"
- 相邻像素方向稍有差异（1°~2°）就会让两组近似平行线产生低频干涉，即摩尔纹
- Structure Tensor 大半径均值可以抑制高频跳变，但平坦区问题依然存在

真正的铜版画也不存在"线条跟随曲面弯曲"——Doré 的线条是固定方向的直线，**靠线宽变化表达明暗**，靠多层不同角度的线叠加表达极暗区。

### 三层叠加策略

| 层 | 角度 | 间距 | 触发范围 | 线宽范围 |
|---|---|---|---|---|
| Layer 1 | 45° | `spacing × 1.00` | 全程（shade 0~1） | 0.06~0.82 × spacing |
| Layer 2 | 135° | `spacing × 1.07` | shade < 0.58 渐入 | 0~0.72 × spacing |
| Layer 3 | 22.5° | `spacing × 0.96` | shade < 0.26 渐入 | 0~0.65 × spacing |

角度选取原则：45°/135°/22.5° 三者不形成整数倍关系；间距 1.00/1.07/0.96 也不成整数比——两个维度同时错位，避免线条节点周期性重叠形成规则网格感。

### 线宽随亮度变化

Layer 1 线宽公式：

```
w1 = mix(0.82, 0.06, shade) × spacing
```

`shade = 0`（纯黑）时线宽为 `0.82 × spacing`，接近实心；`shade = 1`（纯白）时为 `0.06 × spacing`，极细但不消失——高光区保留隐约可见的稀疏细线，避免硬切到纯白。

Layer 2/3 用 `smoothstep` 控制渐入，进入暗区后再按 `enter²` 非线性增粗，让暗部加密速度符合视觉感知。

### 消除节点重叠

当两层线条的节点（交叉点）周期性对齐时，会出现亮度高于周边的规则亮点，视觉上产生网格感。通过两种手段避免：

1. **间距错位**：sp1/sp2/sp3 比值为 1.00/1.07/0.96，使得节点周期不重合
2. **角度非整倍**：45°/135° 虽然互相垂直，但第三层 22.5° 与前两层不成直角，打破对称性

## 辅助函数

### `hatchLayer`

```metal
static float hatchLayer(float2 px, float angle, float spacing, float lineWidth)
```

将像素坐标 `px` 投影到垂直于线方向的轴，取模后判断是否落在线宽范围内。用 `smoothstep` 做半像素宽度的抗锯齿（`aa = 0.5 / spacing`），避免硬锯齿边缘。

`hw = clamp(lineWidth × 0.5, 0, spacing × 0.48) / spacing`：线宽上限为 `0.48 × spacing`，保证相邻线之间始终有至少 `0.04 × spacing` 的间隙，不会粘连成实心块。

### `craftJitter`

```metal
static float craftJitter(float2 px)
```

以 80px 为 cell 的双线性插值 value noise，返回 [0, 1]。用于对每层线宽施加低频扰动，模拟刻刀在不同位置的力道变化（粗细不均匀），提供手工感而不产生细碎颗粒。

扰动幅度：`±18% × spacing`，在中间色调最大，极亮/极暗区用双 `smoothstep` 收窄，避免极细线因扰动断线、极粗线因扰动粘连。

### `paperNoise`

两层 value noise 叠加模拟纸张颗粒：

- 粗粒层：180 cell/uv，占 60%，模拟纸张纤维疏密
- 细粒层：520 cell/uv，占 40%，模拟细微表面起伏

最终对 `paperColor` 施加 `0.92 + noise × 0.10` 的亮度调制，幅度约 ±5%。

## 参数说明

通过 `SCNTechnique` symbols 传入 shader，可在 Swift 端实时更新：

| 参数 | 类型 | 有效范围 | 默认值 | 说明 |
|---|---|---|---|---|
| `hatchSpacing` | float | 3.0 ~ 20.0 | 8.0 | 基础阴影线间距（屏幕像素）。值越小线越密，值越大线越稀。 |
| `contrast` | float | 0.5 ~ 3.0 | 1.5 | 对比度。拉大明暗分层，让线宽变化更戏剧化。 |
| `paletteIndex` | float | 0 / 1 / 2 | 0 | 色板：0=羊皮纸黑白，1=琥珀，2=绿色。 |

UI 控件：侧边栏「线条间距」Slider（3.0~18.0）和「对比度」Slider 直接映射到前两个参数；色板 Picker 映射 `paletteIndex`。

## 色板定义

| 索引 | 名称 | 纸底色 | 墨色 |
|---|---|---|---|
| 0 | 羊皮纸黑白 | `(0.867, 0.831, 0.690)` | `(0.055, 0.048, 0.032)` |
| 1 | 琥珀 | `(0.920, 0.740, 0.380)` | `(0.180, 0.080, 0.010)` |
| 2 | 绿色 | `(0.720, 0.880, 0.740)` | `(0.020, 0.140, 0.060)` |

纸底色同时用作 `sceneView.backgroundColor`，保证 shader 未覆盖的区域（如窗口边缘）颜色一致。

## 深度边缘轮廓

4 方向深度差分（右、上、右上、左上）取最大值，`smoothstep(0.001, 0.004, depthEdge)` 平滑过渡，避免硬阶梯锯齿。相比 Obra Dinn 的 `step` 硬切，`smoothstep` 在边缘过渡带宽约 3 个深度单位，轮廓线更柔和。

## 已探索但放弃的方案

### 动态方向场（Structure Tensor）

在大半径（~32px）内累加 9 个采样点的 `(gx², gxgy, gy²)` 二阶矩，从协方差矩阵取最小特征向量作为等照线方向，让阴影线跟随曲面走向。

放弃原因：
- 平坦面/高光区梯度接近零，Structure Tensor 给出噪声方向，导致平坦区域线条方向突变，出现竖线+横线硬切格子
- 即使对方向场做低通滤波，也无法解决"无信号区域"的根本问题
- 引入 9 次额外纹理采样，性能代价较高

### 间距随亮度变化

让线条间距在亮部宽、暗部窄（间距在 0.55~2.2 倍 spacing 之间按亮度插值），让暗部线条更密。

放弃原因：逐像素的间距变化导致相邻亮度不同的像素处于不同间距的线条坐标系，线条在亮度边界处断裂和扭曲，产生波浪形不连贯感。正确做法是固定间距、动态线宽。

## 后续优化方向

- **多 pass 方案**：如果未来迁移到 `MTKView + SCNRenderer`，可以增加一个独立的亮度平滑 pass，对 sceneColor 做大半径 blur 后再送入 Doré shader，获得比当前逐像素采样更稳定的亮度场，进而支持更精细的方向感知。
- **法线信息**：当前只利用颜色和深度。如果 SceneKit 能输出法线到额外 target，可以用法线直接计算切线方向作为阴影线方向，彻底解决平坦面问题。
- **线条抖动**：目前 `craftJitter` 只扰动线宽。可以进一步对线条的法线方向施加极小角度扰动（±1°~2°），让线条略有弯折感，更接近手工刻线。但需要注意角度扰动会重新引入摩尔纹风险，幅度需严格控制。
